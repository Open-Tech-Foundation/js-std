# memoizeRun

Caches the results of an asynchronous function to improve performance and prevent redundant operations.

<Callout type="info">
  **Single Flight support**: If multiple concurrent calls are made for the same key while the function is still running, they will all share the same pending promise instead of triggering multiple executions.
</Callout>

## Usage

```ts
import { memoizeRun } from "@opentf/std";

const fetchUser = async (id: string) => {
  const response = await fetch(`/api/users/${id}`);
  return response.json();
};

const memoizedFetch = memoizeRun(fetchUser, { maxAge: 5000 });

// First call - triggers fetch
const user1 = await memoizedFetch("123");

// Second call (within 5s) - returns cached promise
const user2 = await memoizedFetch("123");

// Simultaneous calls - only one network request is made
const [uA, uB] = await Promise.all([
  memoizedFetch("456"),
  memoizedFetch("456")
]);
```

## Signature

```ts
function memoizeRun<T, Args extends any[]>(
  func: (...args: Args) => Promise<T>,
  options?: {
    maxAge?: number;
    key?: (...args: Args) => string;
  }
): ((...args: Args) => Promise<T>) & { clear: () => void };
```

### Parameters

- `func`: The asynchronous function to memoize.
- `options`:
    - `maxAge`: Duration in milliseconds before the cache entry expires.
    - `key`: A function to generate a cache key from the arguments. Defaults to `JSON.stringify(args)`.

### Methods

- `clear()`: Clears all cached results for this memoized function.

## Server-Side Benefits

On the server, `memoizeRun` is essential for:
1. **Preventing Cache Stampedes**: By using Single Flight, you protect your database from being overwhelmed when many users request the same non-cached resource at once.
2. **Resource Budgeting**: Using `maxAge` ensures that you don't leak memory with indefinitely stored results while still providing a performance boost for frequent requests.
