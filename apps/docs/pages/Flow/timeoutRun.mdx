> ⏱ Enforces a strict time limit on an asynchronous function.

<Callout type="warning">
  This does not automatically "cancel" the underlying operation (like a network fetch) unless it supports an \`AbortSignal\`. It only ensures the caller doesn't wait indefinitely.
</Callout>

## Syntax

```ts copy
import { timeoutRun } from '@opentf/std';

timeoutRun<T>(
  func: () => Promise<T>,
  ms: number,
  options?: {
    message?: string;
    fallback?: T;
  }
): Promise<T>;
```

## Parameters

- `func`: The asynchronous function to monitor.
- `ms`: The timeout duration in milliseconds.
- `options`:
    - `message`: A custom error message for the timeout rejection.
    - `fallback`: A value to return instead of rejecting when a timeout occurs.

## Returns

A Promise that resolves with the function result or the fallback value.

## Examples

### Basic Usage

```ts
try {
  const data = await timeoutRun(async () => {
    await sleep(3000);
    return 'Done';
  }, 2000);
} catch (error) {
  console.log(error.message); //=> "Operation timed out after 2000ms"
}
```

### With Fallback

```ts
const data = await timeoutRun(async () => {
  await sleep(3000);
  return 'Done';
}, 2000, { fallback: 'Offline' }); //=> "Offline"
```
