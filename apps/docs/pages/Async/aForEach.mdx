> 🔃 Executes an asynchronous callback function for each element of an array.

## Syntax

```ts copy
import { aForEach } from '@opentf/std';

aForEach<T>(
  arr: T[], 
  cb: (value: T, index: number) => Promise<void>
): Promise<void>
```

## Parameters

- `arr`: The source array.
- `cb`: An async function to execute for each element.

## Returns

A `Promise` that resolves when all iterations are complete.

## Examples

```ts
const flatten = async (val) => {
  return new Promise((resolve) => {
    Array.isArray(val) ? resolve(val.flat()) : resolve(val);
  });
};

const nested = [1, 2, 3, [4, 5, [6, 7], 8, 9]];
const tmpArr = [];
await aForEach(nested, async (n) => {
  const val = await flatten(n);
  Array.isArray(val) ? tmpArr.push(...val) : tmpArr.push(val);
});
console.log(tmpArr) //=> [1, 2, 3, 4, 5, 6, 7, 8, 9]
```

## Why?

- **Native `forEach`**: Is not "awaitable". It starts all asynchronous operations simultaneously and completes immediately, which can lead to race conditions if subsequent code depends on the loop's completion.

```js
// Native approach:
arr.forEach(async (x) => await someAsyncFn(x));
console.log('Done'); // Prints before iterations finish.

// @opentf/std approach:
await aForEach(arr, someAsyncFn);
console.log('Done'); // Prints after all iterations finish.
```

- **Execution Order**: `aForEach` ensures that each iteration is fully completed before the next one starts, providing predictable sequential behavior.
