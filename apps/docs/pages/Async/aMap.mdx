> 🗺️ Maps an array to a new array using an asynchronous callback function.

## Syntax

```ts copy
import { aMap } from '@opentf/std';

aMap<T, R>(
  arr: T[], 
  cb: (value: T, index: number) => Promise<R>
): Promise<R[]>
```

## Parameters

- `arr`: The source array.
- `cb`: An async function that returns a `Promise` of the transformed value.

## Returns

A `Promise` that resolves to a new array of transformed values.

## Examples

```ts
const arr = [1, 2, 3, 4, 5];

function multiply(n, i) {
  return new Promise((resolve) => {
    setTimeout(() => resolve(n * i), 10);
  });
}

const result = await aMap(arr, async (n, i) => await multiply(n, i));
console.log(result); //=> [0, 2, 6, 12, 20]
```

## Why?

- **Native `map`**: Returns an array of `Promise` objects (e.g., `[Promise, Promise, ...]`). You would need to manually use `await Promise.all()` to get the actual values. `aMap` handles this resolution internally.

```js
// Native approach:
const results = await Promise.all(arr.map(async (x) => await someAsyncFn(x)));

// @opentf/std approach:
const results = await aMap(arr, someAsyncFn);
```

- **Sequential Execution**: Native `map` with async callbacks starts all operations simultaneously. `aMap` executes them sequentially, which is safer when dealing with rate-limited APIs or database connections.
